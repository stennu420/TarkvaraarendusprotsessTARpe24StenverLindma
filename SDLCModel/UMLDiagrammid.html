UML indeks

UML on modelleerimiskeel,mida kasutatakse tarkvara ja süsteemide
Kirjeldamiseks,planeerimiseks,dokumenteerimiseks ja selleks, et
Arendajad seda ühiselt mõistaksid.



UML ei ole programmeerimiskeel
UML on mõeldud jooniste keelena, sellega saab näitkes:
  • Mida süsteem teeb;
  • Millised klassid ja objektid süsteemis on;
  • Kuidas kasutaja süsteemiga suhtleb;
  • Kuidas objektid omavahel sõnumeid vaheavad;
  • Millistes olekutes süsteem või objekt võib olla.


Kasulikud lingid diagrammide kohta

  • UML Spetsifikatsioon
  • UML Organisatsioon
  • UML Diagrammid




Use Case Diagramm

Use Case diagram on kasutusjuhtude diagramm näitab, kes süsteemi kasutavad
ja mida nad süsteemiga teha saavad.



Class Diagramm

Class diagram on süsteemi staatilise struktuuri kirjeldamise jaoks
Näitab ära millised klassid sõsteemid on,millised atribuudid ja
Meetodid neil klassidel on ja kuidas need klassid omavahel seotud on.



Sequence Diagramm

Sequence diagramm on süsteemi dünaamilise käitumise kirjeldamiseks
Näitab ära kuidas objektid omavahel sõnumeid vahetavad ja
Kuidas need sõnumid järjestikku toimuvad.



State Diagramm

State diagramm on süsteemi või objekti olekute ja nende vaheliste üleminekute kirjeldamiseks
Näitab ära millistes olekutes süsteem või objekt võib olla ja kuidas ta n



Vabalt valitud skeemitüüp



ERD Diagramm

ERD diragram on suhete ja mudelite kirjeldamiseks, mis on seotud andmebaasidega
See näitab ära millised tabelid andmebaasis on, millised väljad neil tabelitel on ja
kuidas need tabelid omavahel seotud on.ERD põhimudel koosneb olemitüüpidest ja see määrab
ära seosed,mis võivad olemite vhel eksisterida



Kus kasutatakse ERD diagramme

ERD diagramme kasutatakse peamiselt andmebaasi disainis nagu MySQL ja PostgreSQL
Seda on ka infosüsteemide analüüsis ja disainis, et mõista andmete struktuuri ja seoseid süsteemis
Tarkvaraarenduses kasutatakse ERD diagramme, et mõista andmete struktuuri ja seoseid süsteemis
ning aidata kaasa andmebaasi disainile
Äriprotsesside modelleerimisel kasutatakse ERD diagramme, et mõista andmete struktuuri ja
seoseid äriprotsessides ning aidata kaasa äriprotsesside disainile



ERD diagrammi elemendid

ERD põhielemendi on Olemid,Atribuddid,Seosed ja Ühenduselemendid
Olemid (Entities) Esindavad objekte nt(Klient ja Toode)
Tavaliselt kujutatakse neid ristkülikutena
Need vastavad sageli andmebaasi tabelitele



Atribuudid

Atribuudid(Attributes) on olemite omadused nt(nimi, ID ja hind)
Neid kujutatakse tavaliselt ovaalidena
Mõned atribuudid võivad olla kohustuslikud,unikaalsed või arvutatavad

Seoased

Seosed(Relatisonships) need näitavad,kuidas olemitest tekivad seosed nt(Klient teeb Tellimuse)
Seal kujuntatakse joonte või rombidena



Ühenduselemendid

Seal näidatakse mitu kirjet võib olla seotud: 1:1, 1:N, N:M
Otsade tähendused(Crow's Foot Notation) näitavad seose tüüpi ja kardinaalsust



ERD Keys(võtmed)

Primary Key

Primary Key ehk primaarvõti sellel on oluline roll andmete täpsuse ja terviklikkuse tagamisel
See on unikaalne identifikaator, mis eristab iga kirjet tabelis
Primary Key peab olema unikaalne ja mitte-null



Foreign Key

Foregin Key ehk võõrvõti lühitalt FK see on ERD atribuut,mis viitab teise tabeli Primary Key-le
Seda kasutatakse kahe tabeli vahel seose loomiseks.See viitab teise tabeli primaarvõtmele
Foreign Key aitab tagada andmete terviklikkust ja võimaldab andmeid tõhusalt seostada



Composite Key

Composite Key ehk koosnevvõti see on ERD atribuut,mis koosneb mitmest atribuudist,
mis koos moodustavad unikaalse identifikaatori
Seda kasutatakse siis, kui ükski atribuut ei ole piisavalt unikaalne,et



Key

Key ehk võti on ERD atribuut,mis on määratud võtmena
See võib olla Primary Key,Foreign Key või Composite Key
Võtmed on olulised andmete täpsuse ja terviklikkuse tagamisel



Koos olemitabelis keyde kasutamine

Erinevaid key-si kasutatakse relatsioonnandmebaasides nagu SQL
tabelite struktuuri tegemiseks ja seoste loomiseks
PK ehk Primary Key idenfitseerib unikaalse rea, FK ehk Foreign Key loob seose
teiste tabelitega,
CK ehk Composite Key koosneb mitmest atribuudist, mis võib olla PK või FK,
mis kasutab mitut veergu
