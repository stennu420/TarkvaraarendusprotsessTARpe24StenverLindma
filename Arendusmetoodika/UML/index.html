Mis on UML?

UML ehk Unified Modeling Language on graafiline keel, See näitab kuidas süsteem või programm töötab.

Kes selle lõid?

UML-i lõid 1990 aastate keskel Grady Booch, Ivar Jacobson ja James Rumbaugh.

Use Case Diagram

Use Case Diagram ehk kasutusjuhtude diagramm kirjeldab süsteemi funktsioone. See näitab, kuidas kasutajad või teised süsteemid tarkvaraga suhtlevad.

Elemendid:
  • Actor ehk tegutseja
  • Use Case ehk kasutusjuht
  • System Boundary ehk süsteemi piir
  • Association ehk seos
  • Include ehk kaasamine
  • Extend ehk laiendamine
  • Generalization ehk üldistamine
[Kasutaja] --> (Logi sisse) [Admin] --> (Halda kasutajaid)

Class Diagram

Class Diagram näitab klasside struktuuri, nende omadusi, meetodeid ja seoseid.

Elemendid:
  • Klassi nimi
  • Atribuudid
  • Meetodid
  • Association ehk seos
  • Inheritance ehk pärimine
  • Composition ehk kompositsioon
------------------ User ------------------ - name - email ------------------ + login() + logout()

Sequence Diagram

Sequence Diagram näitab objektide vahelist suhtlust ajas.

Elemendid:
  • Lifeline
  • Activation bar
  • Message arrow
  • Return message
User -> App: Login request App -> Database: Check credentials Database -> App: Success App -> User: Login confirmed

State Diagram

State Diagram näitab objekti erinevaid olekuid ja nende vahelisi üleminekuid.

Elemendid:
  • Alguspunkt
  • Olek
  • Üleminek
  • Lõppolek
Start -> Pending -> Approved -> Completed

Component Diagram

Component Diagram näitab süsteemi komponente ja nende sõltuvusi.

Elemendid:
  • Komponent
  • Liides ehk interface
  • Sõltuvus ehk dependency
Frontend ---> Backend ---> Database
