UML


Mis on UML?

UML ehk (Unified Modeling Language) on graafiline keel, mida kasutatakse tarkvara ja süsteemide visualiseerimiseks, kavandamiseks ning dokumenteerimiseks. See aitab arendajatel ja ärianalüütikutel keerulisi süsteeme paremini mõista.

UML diagrammide indeks

  • Flowchart / Activity Diagram
  • ERD (Entity Relationship Diagram)
  • Use Case Diagram
  • Class Diagram
  • Sequence Diagram
  • State Diagram
  • Component Diagram

Use Case Diagram

Use Case Diagram ekh (kasutusjuhtude diagramm) see on visuaalne UML-diagramm,mis kirjeldab süsteemi funktsioone. See näitab, kuidas kasutajad või süsteemid kavandatav tarkvaraga suhtleb.

Elemendid:
  • Actot(tegutseja)
  • Use Case (kasutajjuht)
  • System Boundary(süsteemi piir)
  • Association (seos)
  • Include (kaasamine)
  • Extend (laiendamine)
  • Generalization (üldistamine)
[Kasutaja] --> (Logi sisse)
[Admin] --> (Halda kasutajaid)

Class Diagram

Class Diagram näitab klasside struktuuri, nende omadusi ja meetodied ja seoseid

Elemendid:
  • Klassi nimi
  • Atribuudid
  • Meetodid
  • Association
  • Inheritance
  • Composition
------------------ User ------------------ - name - email ------------------ + login() + logout()

Sequence Diagram

Sequence diagram näitab objektide vahelist suhtlust ajas.

Elemendid:
  • Lifeline
  • Activation bar
  • Message arrow
  • Return message
User -> App: Login request App -> Database: Check credentials Database -> App: Success App -> User: Login confirmed

State Diagram

State diagram näitab objekti erinevaid olekuid ja nende vahelisi üleminekuid.

Elemendid:
  • Alguspunkt
  • Olek
  • Üleminek
  • Lõppolek
Start -> Pending -> Approved -> Completed

Component Diagram

Component diagram näitab süsteemi komponente ja nende sõltuvusi.

Elemendid:
  • Komponent
  • Liides (interface)
  • Sõltuvus (dependency)
Frontend ---> Backend ---> Database
